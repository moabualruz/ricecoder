# Code Style and Conventions
- Follow Rust standard naming conventions: snake_case for functions and variables, CamelCase for types and traits.
- Use type hints where appropriate.
- Write docstrings for public functions and types.
- Use rustfmt for formatting.
- Use clippy for linting.
- Organize code into modules.
- Use Result and Option for error handling.
- Follow Rust best practices for async code with tokio.
